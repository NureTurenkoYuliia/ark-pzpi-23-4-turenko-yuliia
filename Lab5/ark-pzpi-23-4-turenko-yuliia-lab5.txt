3 ОПИС ВИКОНАНОЇ РОБОТИ
     Для розгортання створеного проекту було обрано хмарну платформу Microsoft Azure. Цей вибір зумовлений необхідністю забезпечити високу доступність, безпеку та можливість масштабування системи. До основних переваг Azure належать:
* підтримка сучасних версій .NET;
* можливість швидкого розгортання Web API без ручного налаштування серверів;
* вбудована підтримка баз даних (зокрема Azure SQL Database);
* зручні інструменти моніторингу, логування та адміністрування;
* можливість масштабування та подальшого розширення системи.
     Додаток розгорнуто як App Service, що є моделлю «Платформа як сервіс». Це дозволило зосередитися на розробці коду, не витрачаючи час на налаштування серверів, оновлення ОС чи конфігурацію вебсерверів. Azure автоматично забезпечує життєздатність додатку та інтеграцію з середовищем виконання .NET. Таким чином для розгортання було виконано такі етапи:
* створення Web App для розміщення .NET Web API;
* створення Azure SQL Database для зберігання даних;
* налаштування з’єднання між Web App та базою даних через connection string;
* завантаження коду серверної частини з репозиторію та деплой в Azure.
     Замість локального SQL Server було використано хмарне рішення Azure SQL. Воно забезпечує автоматичне керування схемою і безпеку. Міграції бази даних виконуються автоматично при запуску застосунку, що дозволяє підтримувати актуальну структуру БД без ручного втручання.
     Автоматизація через CI/CD (GitHub Actions): проєкт налаштований на автоматичне розгортання. Кожен «пуш» у репозиторій запускає процес збірки та деплою в Azure, що мінімізує людський фактор та пришвидшує доставку нових функцій.
     Після розгортання система працює наступним чином:
* серверна частина постійно доступна через HTTP API;
* IoT-клієнт (ESP32 у симуляторі) підключається до серверу;
* сенсорні дані надсилаються на сервер і зберігаються в базі даних;
* сервер надає команди для IoT-пристрою;
* IoT-клієнт виконує команди та повідомляє сервер про результат.
     Таким чином забезпечується стабільна двостороння взаємодія між сервером і IoT-пристроєм.
     У результаті тестування було підтверджено, що:
* серверна частина стабільно обробляє HTTP-запити;
* дані коректно зберігаються в базі даних;
* IoT-клієнт успішно взаємодіє з сервером у режимі реального часу.
     Система показала достатній рівень стабільності та надійності для демонстраційного IoT-рішення. Обрана архітектура дозволяє масштабувати систему та додавати нові компоненти без суттєвих змін у вже реалізованій логіці.
     Також була розроблена невелика схема взаємодії компонентів програмної системи (рис. Б.1). На схемі показано, як IoT-клієнт складається з окремих компонентів, які взаємодіють між собою та з серверною частиною.

4 ВИСНОВОК
     У результаті виконання лабораторної роботи була успішно досягнута мета з освоєння навичок розгортання програмної системи. Було використано екосистеми Azure, що дозволило створити надійну інфраструктуру, яка забезпечує цілісність даних через реляційні зв'язки SQL Server та надає інструменти для швидкого виявлення й усунення помилок.
